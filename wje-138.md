## 项目主要需求

### 项目概述

【全栈Web应用】车队调度与维护管理平台。面向物流公司和车队管理者，覆盖车辆档案、司机管理、调度派单、油耗统计和维保记录全生命周期管理。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿全栈）

1. **车辆（Vehicle）** — 字段：车牌号、车辆类型（轻卡/中卡/重卡/冷链车/危化车）、品牌型号、购买日期、保险到期日、年检到期日、当前状态（Available/OnTrip/Maintenance/Retired）、累计里程、油箱容量、当前油耗（L/100km）
   - 关联：1辆车 → 多条调度记录、多条维保记录、多条油耗记录
   - 业务动作：添加车辆、更新状态、报废车辆
   - **贯穿全栈**

2. **司机（Driver）** — 字段：姓名、手机号、驾照类型（C1/B2/A2/A1）、驾照到期日、入职日期、当前状态（Available/OnTrip/Leave/Suspended）、累计驾驶时长、违章次数
   - 关联：1个司机 → 多条调度记录
   - 业务动作：添加司机、更新状态、违章记录
   - **贯穿全栈**

3. **调度单（DispatchOrder）** — 字段：单号（自动生成）、车辆ID、司机ID、出发地、目的地、预计出发时间、预计到达时间、实际出发时间、实际到达时间、货物描述、重量（kg）、运费、状态（Pending/Assigned/InProgress/Completed/Cancelled）、创建人ID、备注
   - 业务动作：创建调度单、指派车辆和司机、开始运输、完成运输
   - **贯穿全栈**

4. **维保记录（MaintenanceRecord）** — 字段：所属车辆ID、维修类型（Routine/Repair/Emergency/Inspection）、维修项目列表、维修费用、维修厂、维修日期、下次保养里程/日期、维修状态（Scheduled/InProgress/Completed）
   - **贯穿全栈**

5. **油耗记录（FuelRecord）** — 字段：所属车辆ID、加油日期、加油量（L）、单价、总金额、当前里程、加油站名称、支付方式（Cash/Card/Company）
   - **贯穿全栈**

### 核心页面（≥ 4 个）

1. **调度中心** `/dispatch` — 消费：DispatchOrder + Vehicle + Driver
   - 调度单列表（按状态Tab），创建调度单（下拉选择可用车辆和司机），调度单详情（运输时间线）
   - 使用 `<StatusBadge>`, `<UserAvatar>`, `<Timeline>`

2. **车辆管理** `/vehicles` — 消费：Vehicle + MaintenanceRecord + FuelRecord
   - 车辆列表（卡片+列表双视图），车辆详情（基本信息+维保历史+油耗趋势图+调度记录）
   - 使用 `<StatusBadge>`, `<VehicleCard>`, `<EmptyState>`

3. **司机管理** `/drivers` — 消费：Driver + DispatchOrder
   - 司机列表（含状态筛选），司机详情（基本信息+调度历史+违章记录+驾驶时长统计）
   - 使用 `<StatusBadge>`, `<UserAvatar>`

4. **维保管理** `/maintenance` — 消费：MaintenanceRecord + Vehicle
   - 维保日历视图（即将到期的保养高亮），创建维保记录，维保费用统计
   - 使用 `<StatusBadge>`, `<CalendarCell>`, `<EmptyState>`

5. **油耗分析** `/fuel-analytics` — 消费：FuelRecord + Vehicle
   - 各车油耗趋势折线图、月度总油耗柱状图、异常油耗预警（油耗突增）
   - 使用 `<StatCard>`, `<AlertBanner>`

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / Dispatcher / Driver / Mechanic / Viewer
   - 触达文件层：DB role表 + RBAC中间件 + 路由守卫 + 按钮显隐（Driver只能查看自己的调度单）+ 类型定义

2. **操作日志**
   - 记录调度指派、状态变更、维保确认等关键操作
   - 触达文件层：DB operation_log表 + 日志中间件 + service埋点 + 前端日志页 + 类型定义

3. **全局异常处理**
   - 触达文件层：后端异常过滤器 + controller业务异常 + 前端ErrorBoundary + API拦截器 + 错误码枚举

### 共享枚举/常量（≥ 2 组）

1. **车辆状态 VehicleStatus**：Available / OnTrip / Maintenance / Retired
   - 后端：`backend/src/types/enums.ts`；前端：`frontend/src/types/enums.ts`

2. **调度单状态 DispatchStatus**：Pending / Assigned / InProgress / Completed / Cancelled

3. **维修类型 MaintenanceType**：Routine / Repair / Emergency / Inspection

4. **司机状态 DriverStatus**：Available / OnTrip / Leave / Suspended

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 所有页面
2. **`<UserAvatar>`** — 调度中心、司机管理
3. **`<VehicleCard>`** — 车辆管理
4. **`<Timeline>`** — 调度中心（运输时间线）
5. **`<EmptyState>`** — 所有列表空状态

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useDispatch`** — 封装调度单创建和状态流转逻辑
2. **`usePagination`** — 通用分页
3. **`formatMileage`** / **`formatFuelConsumption`** — 里程和油耗格式化

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色校验
3. **`auditLogMiddleware`** — 操作日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`fleet-dispatch`
- **前端端口**：18708 / 80
- **后端端口**：19208 / 3000
- **前端**：React 18 + TypeScript + Vite
- **UI 库**：Ant Design 5
- **图表**：ECharts
- **后端**：Django + Python（REST Framework）
- **数据库**：PostgreSQL 15
- **认证**：JWT（djangorestframework-simplejwt）
- **状态管理**：Zustand
- **构建工具**：Vite

### 文件结构强制清单

**前端（严禁合并职责到单一文件）**：
```
frontend/src/
├── api/              # vehicle.ts, driver.ts, dispatch.ts, maintenance.ts, fuel.ts
├── stores/           # vehicleStore.ts, driverStore.ts, dispatchStore.ts, maintenanceStore.ts
├── types/            # index.ts, enums.ts, vehicle.ts, driver.ts, dispatch.ts, maintenance.ts, fuel.ts
├── components/common/  # StatusBadge, UserAvatar, VehicleCard, Timeline, CalendarCell, EmptyState
├── hooks/            # useDispatch.ts, usePagination.ts
├── pages/            # DispatchCenter, VehicleManage, DriverManage, MaintenanceManage, FuelAnalytics
├── router/           # index.tsx, guards.tsx
├── utils/            # formatMileage.ts, formatFuelConsumption.ts, request.ts
└── constants/        # apiPaths.ts
```

**后端（严禁合并职责到单一文件）**：
```
backend/
├── fleet_app/
│   ├── views/        # vehicle_views.py, driver_views.py, dispatch_views.py, maintenance_views.py, fuel_views.py
│   ├── serializers/  # vehicle_serializer.py, driver_serializer.py, dispatch_serializer.py, maintenance_serializer.py, fuel_serializer.py
│   ├── services/     # vehicle_service.py, dispatch_service.py, fuel_analytics_service.py
│   ├── models.py     # 所有模型（按实体分段注释）
│   ├── urls.py       # 路由配置
│   ├── permissions.py # RBAC权限类
│   ├── middleware/    # audit_log.py, request_logger.py, error_handler.py
│   └── admin.py      # 后台管理注册
├── config/           # settings.py, urls.py
└── manage.py
```

### 部署要求

Docker Compose 一键部署。

docker-compose.yml 要求：
- 顶层 `name: fleet-dispatch`
- 不写 `version:` 字段
- PostgreSQL 命名卷 + healthcheck
- 后端 depends_on service_healthy
- 前端 `${FRONTEND_PORT:-80}:80`，后端 `${BACKEND_PORT:-3000}:3000`

### Git 要求

`git init` + `.gitignore`。
